Public input boundary

The public input is a constrained GitHub Issue, not arbitrary publishing.

Demo Pages exists to show a bounded workflow surface. A visitor can submit only a constrained GitHub Issue, and the request is processed through GitHub Flows with restricted agent execution rather than through a public editor, CMS, website builder, or user-hosting surface.

The goal is workflow visibility: public intent enters through the issue, and the resulting page remains a subordinate demonstration artifact listed from validated workflow metadata.

Validation boundary

Only machine-validated generated changes may continue toward publication.

Restricted execution: GitHub Flows and a bounded agent profile prepare the change inside the normal SSR and Nunjucks site pipeline.

Required gate: invalid generated metadata or generated templates stop the path before publication.

Normal successful path: validated changes proceed through pull request, merge, GitHub Action deployment, and final production publication without manual owner participation as the normal route.

Meaning boundary: generated pages are not official TeqFW, ADSM, or Alex Gusev editorial content.

What this is not

Demo Pages is not hosting, a CMS, or a publishing entitlement.

It is not user hosting, a website builder, a community publishing area, or an arbitrary content submission mechanism.

Generated content stays inert: no user JavaScript, no user-authored styles, no images, no forms, and no embedded external resources.

The index page is owner-maintained, and generated agents do not edit it directly.
